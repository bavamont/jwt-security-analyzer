Algorithm Notes:

                                    

                                        
HMAC algorithms use symmetric keys (shared secret)

                                        
RSA algorithms use asymmetric keys (private/public key pairs)

                                        
The "none" algorithm disables signature verification (dangerous!)

                                        
Higher bit lengths (384, 512) provide stronger security

                                    

                                

                            

                        


                        

                            

                                
 Attack Vectors Guide

                            

                            

                                

                                    

                                        
Algorithm None Attack:

                                        
Removes signature verification by changing the algorithm to "none". Test if your application validates algorithms.

                                    


                                    

                                        
Algorithm Confusion:

                                        
Changes RS256 to HS256, using the public key as an HMAC secret. Exploits applications that don't properly validate algorithm types.

                                    


                                    

                                        
Weak Secret Attack:

                                        
Targets HMAC-signed JWTs with predictable secrets. Use dictionary attacks to discover weak passwords like "secret", "password", or "key".

                                    


                                    

                                        
Token Replay Attack:

                                        
Exploits tokens without proper expiration or time-based claims. Removes exp, nbf, and iat claims to enable indefinite token reuse.

                                    


                                    

                                        
JWK Injection:

                                        
Injects a malicious JWK (JSON Web Key) in the token header to bypass signature verification.

                                    


                                    

                                        
Kid Parameter Injection:

                                        
Exploits the "kid" (Key ID) parameter for path traversal and injection attacks when applications use it unsafely.

                                    


                                    

                                        
JKU URL Hijacking:

                                        
Points the "jku" parameter to attacker-controlled servers hosting malicious JWK sets for signature bypass.

                                    


                                    

                                        
X5U Certificate Exploit:

                                        
Exploits the "x5u" parameter by pointing to malicious X.509 certificate chains, enabling signature bypass and SSRF attacks.

                                    


                                    

                                        
JWT Smuggling:

                                        
Exploits parsing differences between JWT libraries to smuggle malicious tokens.

                                    


                                    

                                        
Nested JWT Attack:

                                        
Places one JWT inside another's payload to exploit validation logic and achieve privilege escalation through token confusion.

                                    


                                    

                                        
Audience Confusion:

                                        
Exploits weak audience claim validation to enable cross-service token reuse and privilege escalation.

                                    


                                    

                                        
Parameter Pollution:

                                        
Uses duplicate JSON keys with conflicting values to exploit parsing differences between libraries.

                                    


                                    

                                        
Timing Attack:

                                        
Analyzes signature verification timing to extract key material or bypass security through statistical timing analysis.

                                    


                                    

                                        
JWT Sidejacking:

                                        
Targets JWT-specific session management vulnerabilities including token hijacking, fixation, and cross-site theft.

                                    


                                    

                                        
JWKS Cache Poisoning:

                                        
Poisons JSON Web Key Set caches with malicious keys to compromise all applications using the poisoned cache.

                                    


                                    

                                        
Quantum-Prep Attack:

                                        
Evaluates JWT resistance to future quantum computing attacks and recommends post-quantum cryptography migration.

                                    

                                

                            

                        


                        

                            

                                
 Tools Guide

                            

                            

                                

                                    

                                        
JWT Decoder:

                                        
Paste any JWT token to see its header, payload, and signature in human-readable format. Analyzes claims and provides token structure insights.

                                    


                                    

                                        
JWT Encoder:

                                        
Create custom JWT tokens by specifying header, payload, algorithm, and secret. Supports templates for common token structures.

                                    


                                    

                                        
Security Analysis:

                                        
Comprehensive vulnerability scanner that checks for common JWT security issues and provides a security score.

                                    


                                    

                                        
Brute Force:

                                        
Test HMAC secret strength using dictionary attacks. Load common secrets or custom wordlists to crack weak tokens.

                                    


                                    

                                        
Key Generator:

                                        
Generate cryptographically secure HMAC secrets and RSA key pairs for production use. Supports multiple key sizes.

                                    


                                    

                                        
Token Validator:

                                        
Verify JWT token signatures and validate claims against provided secrets or public keys.

                                    


                                    

                                        
Attack Vectors:

                                        
Generate 16+ different attack payloads to test JWT vulnerabilities including algorithm confusion, JWK injection, and parameter pollution.

                                    


                                    

                                        
Network Proxy:

                                        
Intercept HTTP/HTTPS traffic to capture JWT tokens from live network requests. Supports SSL/TLS decryption with CA certificate installation.

                                    


                                    

                                        
Replay Attack Simulator:

                                        
Test JWT tokens for replay vulnerabilities by sending multiple requests with the same token to target endpoints.

                                    


                                    

                                        
Token Comparison:

                                        
Compare two JWT tokens side-by-side to identify differences in headers, payload claims, and signatures.

                                    


                                    

                                        
HTTP Tester:

                                        
Test JWT tokens in real HTTP requests. Configure headers, body, and authorization to see how tokens behave in API calls.

                                    


                                    

                                        
Base64 Tools:

                                        
Encode plain text to Base64 or decode Base64 strings back to plain text. Useful for manually crafting JWT components.
